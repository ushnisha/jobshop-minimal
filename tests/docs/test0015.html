Test 0015:

  • This test has the two demands, one for each part, both due on 2018-01-14, and both for 270 units (base scenario data without any changes).
  • The demand for PART001 has a priority of 1, while the demand for PART002 has a priority of 2. That is, the demand for PART001 has a higher priority and is planned first. We will see the effect of this in future test plans.
  • The RESOURCE_CONSTRAINED planning parameter is set to TRUE.

Variations from the Base Scenario:

  • The RESOURCE_CONSTRAINED planning parameter is set to TRUE.

The expected result is as follows:

  • Both demands are satisfied on time
  • We schedule OP-10 and OP-20 for both parts at the same time but on different workcenters. OP-10 for PART001 is planned on the primary workcenter, RES001 while OP-10 for PART002 is planned on RES002, the alternate workcenter. Similarly, OP-20 for PART001 loads RES003 and OP-20 for PART002 loads RES004.
  • This is a reflection on the default preference of the solver to plan JIT on the alternate workcenter rather than to build ahead on the primary workcenter.
